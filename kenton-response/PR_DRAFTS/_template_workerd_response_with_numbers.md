These are measurements, not a patch. For the actual V8 change:

- `src/v8-cow/RFC.md` — design for v8-dev@ covering hook points,
  scope, open questions (pointer identity, sandbox cage, snapshot
  versioning).
- `src/v8-cow/cow_builtins_prototype.cc` — standalone C++ prototype of
  the shared-slab + promote data structure (no V8 dep). Validates
  that the sharing+COW shape is not pathological on synthetic data
  (~99.57% savings at 1000 isolates with 10% writing; see
  `results/p3_prototype.txt`).
- `src/v8-cow/patch_skeleton.diff` — V8 patch that adds
  `CowSharedBuiltins` class + flag, applies cleanly against V8 HEAD,
  with `UNIMPLEMENTED()` at the `PromoteOnWrite` site by design.
- `src/v8-cow/hooks_reference.md` — the four edits in existing V8
  files that would wire it in (line numbers drift per V8 revision so
  these are documented rather than patched).

The full patch is the ~4-8 week job. Before I commit that time, I'd

## What I'm no longer claiming

  workerd isolate overhead per `results/p3_memory.json` is
